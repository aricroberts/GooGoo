Project: goo-goo
ぐーぐー — "growl growl," the sound your stomach makes

Eat on your body's schedule, not the clock's. goo-goo plans meals around your real energy rhythm — starting as a simple planner today, building toward a system that reads your biometrics, books the food, and times the delivery so it never spoils and never comes too late.

Concept — not yet live OVA v0.3 — "The Renaming" Wellness only — not medical advice Private by design
This page is a working concept, not a live tool. Formerly floated under the working name "Even." — now tracked as Project: goo-goo, versioned like an anime series: builds ship as Seasons made of Episodes, and non-canon reframes (like this rename) ship as an OVA. It exists to pressure-test one idea — that the biggest lever on daily energy is *when* you eat, not just what — and to show how a full biometric-to-delivery loop could be built one honest episode at a time, instead of promising the finale on day one.

How reminders work — S1E2, "The Call"

Pick one real-time channel to avoid a wall of duplicate alerts for the same meal. Each has a real drawback — shown here, not hidden.

Calendar push Real-time
Your schedule is written as calendar events with alerts — Google/Apple Calendar's own notifications fire it. No infrastructure of ours.
Drawback: only as reliable as your own calendar notification settings.
SMS / Email Real-time or digest
A message fires at the set time. Works on any device, no install.
Drawback: a message, not a system notification — easy to miss; real per-message cost at scale.
Web push Real-time
A real OS-level push notification from the site. Works from an open tab on Android Chrome.
Drawback: on iOS, only works after "Add to Home Screen" — and doesn't fire at all for EU users.
The hierarchy — no tornado of alerts
One real-time channel. Digest and escalation are optional add-ons, not duplicates.
Exactly one real-time channel is active per meal window (Calendar push, SMS, or Web push) — picking a second doesn't stack it, it replaces the active one.
Email can run alongside any real-time channel as a once-a-day morning digest of the full schedule, not a per-meal ping — so it never duplicates the real-time alert.
Escalation is opt-in and single-fire: an unacknowledged alert triggers one fallback via a second channel, never a repeating retry.
Opening any alert marks that window as seen, so the digest or escalation won't re-announce it.
Season 1 — full episode list

"The Growl" arc. No app required for any episode in this season.

S1E1
The GrowlWeb tool
Rule-based weekly schedule + shopping list from manual input. No app, no device, no login.
S1E2
The CallWeb tool
Calendar push, SMS/email, or web push — user's choice, with the hierarchy above to prevent duplicate alerts.
S1E3
The DeliveryWeb tool
Recipe matching landed early in S1E1: a 24-recipe embedded starter set (not a live Spoonacular call — no backend to hold that key) is filtered by diet style and allergy tags, matched into each cook slot, and aggregated into a specific-ingredient shopping list. What's still ahead here: a live recipe database, Instacart's Developer Platform API pre-building a real cart or shopping list, and Google Places/Yelp surfacing nearby delivery or pickup options. Reminders would link to a one-tap checkout — you'd still confirm and pay.
S1E4
SyncWeb tool
The schedule lives on the household's real Google or Apple calendar — and doubles as the calendar-push reminder channel above.

Reality check (why it's paced this way)
There's no cloud API for glucose data
Apple Health and Google Health Connect only live on-device. "Awakening" (S2E1) needs an actual phone app to read them — this can't stay a web page for that part.
CGM access is gatekept — mostly
Prescription CGMs require a business partnership. Over-the-counter sensors (like Stelo) sync to Apple Health/Health Connect with no approval needed — the realistic entry point, if Season 2 is ever greenlit.
"Boost glucose" is medical-sounding
Framed as diabetes management, this needs clinical validation. Framed as energy and focus, it stays wellness — same lane as the rest of the site.
Delivery is more real than it sounds
Part of S1E3 is already live: S1E1 matches recipes and turns them into a specific-ingredient shopping list, no live API needed. The part that's still a promise is the cart — Instacart's Developer Platform API already supports self-serve cart and shopping-list creation built for third-party integration, but wiring it in needs a backend to hold the key.
This page describes a concept only. No data is collected, no wearables are connected, and nothing here is medical advice or a diabetes management tool. Any future version dealing with real glucose data would need its own privacy review beyond the rest of this site.
